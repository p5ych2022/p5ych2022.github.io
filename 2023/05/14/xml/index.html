    可扩展标记语言（英语：Extensible Markup Language，简称：XML），是一种标记语言（类似于HTML）。XML是从标准通用标记语言（SGML）中简化修改出来的。

    XML没有使用预定义的标记，因此使用者可以根据自己的设计需求定义专属个人的标记。这是一种十分有用的可存储、可搜索和可共享的格式存储数据的方法。最重要的是，因为 XML 的基本格式是标准化的，所以如果您跨系统或平台共享或传输 XML，无论是在本地还是在互联网上，接收方仍然可以根据标准化的 XML 语法解析数据。

    XML被设计用来传输和存储数据，不用于表现和展示数据，HTML则用来表现数据

    XML用途

    • XML把主句从HTML中分离 （数据存储在独立的XML文件中，通过js读取）
    • XML简化数据共享（纯文本格式存储方式）
    • XML简化数据传输 （XML可以通过各种不兼容的应用程序来读取数据）
    • XML 简化平台变更
    • XML 使您的数据更有用
    • XML 用于创建新的互联网语言（很多新的互联网语言都是XML实现的）

    XMLHttpRequest

    XMLHttpRequest（XHR）对象用于与服务器交互。通过 XMLHttpRequest 可以在不刷新页面的情况下请求特定 URL，获取数据。这允许网页在不影响用户操作的情况下，更新页面的局部内容。XMLHttpRequest 在 AJAX 编程中被大量使用。

    XHR就是一个浏览器的接口。而XHR对象用于幕后与服务器交互数据。通俗一点就是一种拿取数据的规则

    CDATA

    PCDATA

    被解析的字符数据

    XML 文档中的所有文本均会被解析器解析。

    (当某个元素被解析时，它里面的文本一起被解析【可能有子元素在里面】)

    只有 CDATA 区段中的文本会被解析器忽略。

    CDATA

    （未解析）字符数据

    为什么设置CDATA?因为有一些特殊字符【< > &】被解析之后可能破坏xml的格式,发生错误。而某些文本，比如 JavaScript 代码，包含大量 “<” 或 “&” 字符。为了避免错误，可以将脚本代码定义为 CDATA。

    语法

    CDATA 部分由 “<![CDATA[**” 开始，由 “**]]>“ 结束

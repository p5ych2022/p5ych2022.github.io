    PHP（PHP: Hypertext Preprocessor）即超文本预处理器，是在服务器端执行的脚本语言，尤其适用于Web开发并可嵌入HTML中。PHP语法学习了C语言，吸纳Java和Perl多个语言的特色发展出自己的特色语法，并根据它们的长项持续改进提升自己，例如java的面对对象的编程，该语言当初创建的主要目标是让开发人员快速编写出优质的web网站。PHP同时支持面向对象和面向过程的开发，使用上非常灵活。 PHP是世界上最好的语言捏

    预定义变量

    又叫超全局变量，是系统已经定义的变量，可以直接调用

    常见的有：

    • $GLOBALS:$GLOBALS 是一个包含了全部变量的全局组合数组。变量的名字就是数组的键。
    • $_SERVER$_SERVER 是一个包含了诸如头信息(header)、路径(path)、以及脚本位置(script locations)等等信息的数组

     

    • $_REQUEST：用于收集HTML表单提交的数据。
    • $_POST：$_POST 被广泛应用于收集表单数据，在HTML form标签的指定该属性："method="post"。
    • $_GET：$_GET 被广泛应用于收集表单数据，在HTML form标签的指定该属性："method="get"。$_GET 也可以收集URL中发送的数据。
    • $_FILES：$_FILES被广泛用于收集表单数据，接收post上传的文件数据 关于表单：
           表单中的name属性对应后端数组的key属性
       例如表单中有：
           name="fname"
       则接收数据要用：
           $name = $_REQUEST['fname'];
    • $_COOKIE：$_COOKIE用于取回 cookie 的值。
    • $_SESSION：session：PHP session 变量用于存储关于用户会话（session）的信息，或者更改用户会话（session）的设置。Session 变量存储单一用户的信息，并且对于应用程序中的所有页面都是可用的。存储和取回 session 变量的正确方法是使用 PHP $_SESSION 变量

    文件上传

    修改配置文件

    实现文件上传功能，首先要在php.ini中开启文件上传，并对其中的参数作出合理的设置，找到uploads项，可以看到下面三个属性值，其含义如下：

    file_uploads:如果是on，则说明服务器开启了文件长传功能；如果为off则关闭了文件上传功能；

    upload_tmp_dir:上传文件的临时目录，在文件被成功上传之前，文件首先会存放到服务器的临时目录中，不设置的为系统默认的目录

    upload_max_filesize：服务器允许上传的文件的最大值，以MB 为单位，系统默认为2MB。

    还有几个属性也会影响：

    max_excution_time:PHP中一个指令所能执行的最大时间，单位是秒

    memory_limit:PHP中一个指令所分配的内存空间，单位是MB

    1.相同文件名的文件上传会存在文件覆盖（所以需要对文件进行重命名）

    2.任意文件上传（对文件后缀，mime_typr，内容进行过滤）

    会话机制

    为什么会有会话机制？

    会话指浏览器和服务器之间的交流（前端和后端的交流）

    浏览器/服务器

    浏览器（客户端）：Chrome/Firefox/IE/edge

    服务器（服务端）：Apache/nignx/IIS

    Apache

    Apache(音译为阿帕奇）是世界使用排名第一的Web服务器软件。它可以运行在几乎所有广泛使用的计算机平台上，由于其跨平台和安全性被广泛使用，是最流行的Web服务器端软件之一。它快速、可靠并且可通过简单的API扩充，将python/perl等解释器编译到服务器中。

    有了Apache，我的电脑才能当做服务器来跑

    基于BS架构的系统：

    B：browser 浏览器

    S : server 服务器

    客户端和服务端之间传输需要一个协议：http协议/https协议

    PHP面向对象

    概念的引入

    面向对象？万物皆对象;接触到的所有东西都是对象，任何事物都有分类。

    为什么学面向对象？

    1.需要具备面向对象的编程思维

    2.需要能看懂面向对象的语法 =》代码审计

    3.相关漏洞需要面向对象的知识 =》反序列化

    面向过程与面向对象

    面向对象更方便，效率更低，消耗内存大，拓展性好

    面向过程更繁琐，效率更高，消耗内存更低，拓展性差
